Without matter the used language in the server, any one needs to have the following list of features

- Pipe: To validate the data contained in the request
- Guard: Make sure the user is authenticated
- Controller: Route a request to a particular function
- Service: Run a specific business logic
- Repository: Access the database

Inversion of control principle: Classes should not create instances of its dependencies of its own
  - In a morealistic explanation, the idea is to inject the mentioned dependency inside class constructor, but with a twist
  - Instead injecting a specfic and defined class (a MySql instance), it uses a generic interface with the same properties and methods as any possible possibility
  - That technique will help the code to implement very different implementations as injections of that class, but they have to comply with parent class interface.
  - This also will help on testing scenarios, where can be used mocked implementations to avoid undesirable effects (slow running, random problems with external services among others)
